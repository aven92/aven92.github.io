# 6. 结论
## (1) Shard Server 故障

1) 分片节点故障，可以按照 MongoDB 副本集故障测试和解决方案中解决，仍可以实现可用性

2) 整个分片故障，如果可以重新启动的话，仍然可以使用，否则只能从备份中恢复，必然会丢数据

## (2) Config Server 故障

只要有可用的配置服务器，整个集群都可以正常使用

## (3) Query Routers 故障

只要有可用的 mongos 服务器，整个集群就可以正常使用，要客户端也实现 failover，需要将代码写成连接多台 mongo 的方式
